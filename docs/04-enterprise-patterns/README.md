# Enterprise Patterns

This directory is reserved for patterns such as DTO, Repository, Specification, Domain Event, Outbox, Inbox, Retry, Circuit Breaker, and similar application-scale concerns.

The MVP references some of these ideas in scenario and anti-pattern guidance, but full standalone guides are intentionally deferred to keep the first version focused.

Future additions should stay problem-first and should not present enterprise patterns as default architecture for every project.
