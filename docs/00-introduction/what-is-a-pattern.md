# What Is a Pattern?

A software pattern is a reusable solution shape for a recurring design
problem. It is not a law, a framework, or a guarantee of good
architecture by itself.

## What a Pattern Gives You

- Shared language for discussing a recurring problem
- A known trade-off profile
- A starting point for reasoning about structure

## What a Pattern Does Not Give You

- Automatic correctness
- Free maintainability
- Permission to ignore the simpler option

## Practical Rule

If you cannot describe the recurring problem first, you do not need the pattern yet.
