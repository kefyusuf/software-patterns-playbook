# How To Read This Repository

This repository is meant to be used by problem shape, not by pattern loyalty.

## Recommended Path

1. Start with the decision guides if you know the problem but not the pattern name.
2. Read the pattern guide when you want trade-offs, placement, and misuse warnings.
3. Read a scenario guide when you want to see multiple patterns working together.
4. Read the anti-pattern guides when a design feels overbuilt or unclear.

## Use It During Real Work

- Before implementing a new abstraction
- During architecture review
- During pull request review
- When refactoring a growing service or integration boundary

## What To Ignore

Do not treat roadmap items or future slices as mandatory architecture for your project. Use only the parts that solve the current problem.
